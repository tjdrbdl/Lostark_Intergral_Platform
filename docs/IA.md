# Information Architecture

최종 갱신: 2026-07-13

## 핵심 사용자 흐름

### 원정대 운영

`/` 검색 → `/expedition/[name]` 원정대 투자 후보 → `/weekly/[name]` 주간 보조

### 레이드 조율

`/raids/new` 모집방 생성 → `/raids/[code]` 참여·가능 시간 제출 → `/raids/[code]/manage` 일정 확정

## 라우트 책임

| 라우트 | 책임 | MVP 우선순위 |
|---|---|---|
| `/` | 제품 가치 설명, 캐릭터 검색, 저장 원정대 재진입 | P0 |
| `/search?q=...` | 검색 입력 정규화와 원정대 결과 이동 | P0 |
| `/expedition/[name]` | 원정대 요약, 캐릭터 비교, ROI 후보와 근거 | P0 |
| `/character/[name]` | 선택 캐릭터의 상세 근거 확인 | P1 |
| `/weekly/[name]` | 레벨 기준 가능 콘텐츠와 투자 후속 후보 | P1 |
| `/raids/new` | 초대형 레이드 모집방 생성 | P1 |
| `/raids/[code]` | 초대 참여, 캐릭터·역할·가능 시간 제출 | P1 |
| `/raids/[code]/manage` | 방장용 참여 현황과 최종 일정 확정 | P1 |

## 내비게이션 원칙

- 검색 결과의 기본 단위는 캐릭터가 아니라 원정대다.
- 주간 페이지는 실제 완료 추적기가 아니라 콘텐츠·투자 판단 보조임을 표시한다.
- 레이드 모집방은 공개 검색에 노출하지 않고 초대 링크로만 접근한다.
- 데이터 출처와 신뢰 상태는 원정대·캐릭터·주간 화면에서 동일한 형식으로 표시한다.
