# 초대형 주간 레이드 모집 MVP

## 목적

친구, 길드원, 고정 파티처럼 이미 함께 갈 의사가 있는 사람들이 메신저에서 여러 번 시간을 묻지 않고 레이드와 시간을 확정하게 돕는다.

## 핵심 가설

초대 링크 하나에서 참가 캐릭터와 가능한 시간을 모으면, 메신저 투표만 사용할 때보다 적은 왕복으로 주간 레이드 일정을 확정할 수 있다.

## 사용자 역할

### 방장

- 모집방 생성
- 레이드와 난이도 선택
- 모집 인원과 후보 시간 1~5개 설정
- 초대 링크 공유
- 참여 현황 확인
- 최종 시간 확정 또는 방 취소

### 참여자

- 초대 링크로 입장
- 표시 이름과 캐릭터명 입력
- 역할 또는 포지션 선택
- 가능한 후보 시간 선택
- 참여 정보 수정 또는 참여 취소

## 최소 데이터

### RaidRoom

- 초대 코드
- 레이드, 난이도, 모집 인원
- 후보 시간 목록
- 상태: 모집 중, 확정, 취소
- 최종 시간
- 생성·수정 시각
- 방장 수정용 비밀 토큰

### RaidParticipant

- 모집방 ID
- 표시 이름, 캐릭터명
- 직업과 아이템 레벨: API 조회 성공 시 보조 표시
- 역할/포지션
- 가능한 시간 목록
- 참여 상태
- 생성·수정 시각

## 접근 정책

- 공개 목록과 검색은 제공하지 않는다.
- 초대 코드를 아는 사람만 방을 볼 수 있다.
- 로그인 없이 참여할 수 있다.
- 방장 변경은 별도의 수정용 비밀 링크로 보호한다.
- 민감한 개인정보는 수집하지 않는다.

## 화면

### 모집방 생성

레이드, 난이도, 인원, 후보 시간을 입력하고 초대 링크를 발급한다.

### 참여 화면

현재 참가자, 시간별 가능 인원, 내 캐릭터와 가능한 시간을 한 화면에서 입력한다.

### 방장 관리

시간별 가능 인원과 역할 구성을 비교하고 최종 일정을 확정한다.

## 제외 범위

- 공개 랜덤 매칭
- 실시간 채팅·음성
- 친구·길드 소셜 그래프
- 자동 스펙 심사와 파티 조합
- 대기열, 신고, 평판, 제재 시스템
- 푸시 알림, Discord 봇, 캘린더 연동
- 실제 레이드 완료 추적

## 완료 조건

- 새 모집방 생성과 초대 링크 발급
- 다른 브라우저에서 링크로 참여
- 참여 정보 생성·수정·취소
- 시간별 가능 인원 집계
- 방장의 최종 일정 확정
- 만료되거나 취소된 방의 명확한 상태 표시
- 참가자 API 조회 실패가 방 참여 자체를 막지 않음

## 구현 전 결정 사항

- 영속 저장소와 보존 기간
- 초대 코드 길이와 충돌 정책
- 방장 비밀 토큰 저장 방식
- 익명 참여 수정 권한 방식
- 레이드·난이도 기준 데이터의 갱신 주체
