# 문서 안내

## 문서 우선순위

1. 사용자의 현재 요청
2. `PRD.md` — 현재 MVP 범위와 성공 기준
3. `RAID_PARTY_MVP.md` 또는 `API_SCHEMA.md`
4. `IA.md` — 핵심 흐름과 라우트 책임
5. `sprints/current/*` — 현재 Sprint 구현 지시와 QA
6. `sprints/archive/*` — 과거 핵심 기록

## 개발 이력

현재 Sprint 구현 지시는 `sprints/current/SPRINT7_HANDOFF.md`만 사용하고, 검증 결과는 `sprints/current/SPRINT7_QA.md`에 유지합니다. `sprints/archive/`에는 핵심 과거 기록만 보관합니다.

## 문서 원칙

- 현재 사실과 향후 계획을 구분합니다.
- 실제 API 데이터와 샘플·추정 데이터를 구분합니다.
- 완료된 기능보다 검증된 사용자 가치를 기준으로 MVP를 판단합니다.
- 새로운 기능은 반드시 제외 범위와 성공 기준을 함께 기록합니다.
